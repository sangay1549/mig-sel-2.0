# OpenCode Conversation — 2026-05-23

## Context

User requested changes to the complaint merging system in the mig-sel app (React 19 + Vite + TypeScript + Supabase).

## Changes Made

### 1. Merge Rules Updated (`complaint-monitor.tsx`)

- **masterOptions** now includes ALL complaints with `parent_id === null` (removed `status !== 'resolved'` and `!childrenMap.has(c.id)` filters)
- **pending** complaints can merge to any status (pending, in-progress, resolved)
- **in-progress** complaints can merge to in-progress or resolved
- **resolved** complaints still cannot be linked (button disabled)
- Masters (complaints with children) can now be merged into other complaints (reversible)

### 2. Grandchildren Reassignment (`handleMerge` in `complaint-monitor.tsx`)

- When a master complaint (one with children) is merged into another master, all its children are reassigned to the new master
- This preserves the status cascade: changing the new master's status updates all merged complaints

### 3. Merge Modal — Photos & Details (`complaint-monitor.tsx`)

- Each master option in the merge modal now shows:
  - Photo thumbnail (or "No img" placeholder)
  - Title + description (2-line clamp)
  - Category, urgency, and status badges
- Admin can identify complaints visually before choosing the master

### 4. Admin Dashboard — Photo Click Shows Details (`complaint-monitor.tsx`)

- Clicking a complaint's image in the table opens a **Complaint Details** modal
- Modal shows: image (clickable → fullscreen via ImageLightbox), title, description, category/urgency/status badges, location
- If complaint is a **child**: shows parent info with amber background
- If complaint is a **parent**: shows list of merged children with their thumbnails and descriptions
- Clicking the image within the detail modal toggles fullscreen

### 5. Map — Hide "Better Quality" for Parents (`grievance-map.tsx`)

- Added `!childrenMap.has(g.id)` to the "Your photo has been merged — better evidence" notification condition
- Parent complaints (those with children) no longer show this notification even if they're merged as a child

## Key Files Modified

- `src/features/complaint/components/complaint-monitor.tsx` — merge logic, masterOptions, merge modal with photos, complaint preview modal
- `src/features/auth/grievance/components/grievance-map.tsx` — hide notification for parents
